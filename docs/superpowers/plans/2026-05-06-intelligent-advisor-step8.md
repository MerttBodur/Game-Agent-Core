# Intelligent Advisor — Step 8 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Activate the popularity signal end-to-end. Land `games.json` (target ≥80 entries, 150 ideal), build `popularity.json` from it, merge popularity onto tools at module load, flip `DATASET_IS_THIN` to `false`, append per-game RAG chunks, re-seed embeddings, and walk through the 5 canonical scenarios from spec §7 day 3 to confirm the demo works end-to-end.

**Architecture:** Manual data entry has been running in parallel across days 1–3 (~4h total). The dataset lives as JSON next to the tool catalog. A small build script (`buildPopularityFromDataset.ts`) computes per-engine + per-language ratios across the five archetype scopes and writes `popularity.json` (a sibling, not a codegen rewrite of `gameDevTools.ts` — keeps the source diff-clean across dataset updates). `gameDevTools.ts` imports `popularity.json` at module load and merges values onto the catalog by tool name. The dataset also feeds new RAG chunks (one per game) for the Reality Check panel and the LLM's `mismatchReasons` text.

**Tech Stack:** Node.js (ESM), TypeScript, esbuild bundle, pgvector seeded via the existing `seedRag.ts`.

**Project conventions (read first):**
- All commands run in PowerShell.
- The dataset's primary purpose is **demo stability**, not statistical rigor. ≥80 entries with reasonable archetype balance is sufficient for v1.
- `popularity.json` is **generated** — never edit by hand.
- `games.json` is the source of truth.
- Run `pnpm --filter @workspace/api-server run rag:seed` after dataset changes that should be retrievable.
- Don't add a DB table for games — the popularity merge happens at module load, not at request time.

**Anti-overengineering boundary:**
- No SQL ingestion of `games.json`. JSON file → in-memory dict at startup. Period.
- No CSV import. JSON only.
- No `popularity.json` schema validator. The build script wrote it; the loader trusts it.
- No CLI for "add a game" — Markdown editing of `games.json` is the workflow.
- The merge by tool name uses a case-insensitive direct map. No fuzzy matching.
- Don't backfill `popularityByArchetype` to the DB seeded `tools` table — the catalog merge happens server-side at module load; the DB table is a `toolId` lookup only.

- [ ] **Step 2.1: Create the file**

Write the full content of `artifacts/api-server/src/scripts/buildPopularityFromDataset.ts`:

```ts
import { readFileSync, writeFileSync } from "node:fs";
import { resolve } from "node:path";

type Scope = "jam" | "prototype" | "indie" | "AA" | "AAA";

interface GameEntry {
  title: string;
  archetype: Scope;
  engine: string;
  language: string;
}

const SCOPES: Scope[] = ["jam", "prototype", "indie", "AA", "AAA"];

const datasetPath = resolve(process.cwd(), "src/lib/games-dataset/games.json");
const outputPath = resolve(process.cwd(), "src/lib/games-dataset/popularity.json");

const raw = readFileSync(datasetPath, "utf8");
const games = JSON.parse(raw) as GameEntry[];

const counts: Record<string, Record<Scope, number>> = {};
const totalsByScope: Record<Scope, number> = { jam: 0, prototype: 0, indie: 0, AA: 0, AAA: 0 };

function bump(toolName: string, scope: Scope): void {
  const key = toolName.trim();
  if (!key) return;
  if (!counts[key]) counts[key] = { jam: 0, prototype: 0, indie: 0, AA: 0, AAA: 0 };
  counts[key][scope] += 1;
}

for (const g of games) {
  if (!SCOPES.includes(g.archetype)) continue;
  totalsByScope[g.archetype] += 1;
  bump(g.engine, g.archetype);
  bump(g.language, g.archetype);
}

const popularity: Record<string, Record<Scope, number>> = {};
for (const [tool, vec] of Object.entries(counts)) {
  const ratios: Record<Scope, number> = { jam: 0.5, prototype: 0.5, indie: 0.5, AA: 0.5, AAA: 0.5 };
  for (const scope of SCOPES) {
    const total = totalsByScope[scope];
    ratios[scope] = total > 0 ? vec[scope] / total : 0.5;
  }
  popularity[tool] = ratios;
}

writeFileSync(outputPath, JSON.stringify(popularity, null, 2) + "\n", "utf8");
console.log(`Wrote popularity.json — ${Object.keys(popularity).length} tools across ${games.length} games.`);
```

Notes:
- The default `0.5` for an unobserved scope is the neutral value (no signal). Combined with Step 6's `(p - 0.5) * range` formula, that is a zero-delta contribution.
- Tool names are matched **exactly** to entries in `gameDevTools.ts`. Spelling discipline is enforced at data-entry time (README note in Step 1.1).

- [ ] **Step 4.2: Add `buildGameDocuments` and merge into the seed pipeline**

In that file, add:

```ts
import gamesData from "../games-dataset/games.json" with { type: "json" };
// or: const games = JSON.parse(readFileSync(...));

interface GameEntry {
  title: string;
  archetype: "jam" | "prototype" | "indie" | "AA" | "AAA";
  engine: string;
  language: string;
  year: number;
  budgetUSD?: number | null;
  teamSize?: number | null;
  devYears?: number | null;
  source?: string;
}

export function buildGameDocuments() {
  const games = gamesData as GameEntry[];
  return games.map((g) => ({
    id: `game::${slugify(g.title)}::${g.year}`,
    content: gameToChunkText(g),
    metadata: {
      title: g.title,
      archetype: g.archetype,
      engine: g.engine,
      language: g.language,
      year: g.year,
      source: g.source ?? "",
      kind: "game",
    },
  }));
}

function gameToChunkText(g: GameEntry): string {
  const parts = [
    `${g.title} (${g.year}) — ${g.archetype} project.`,
    `Engine: ${g.engine}. Language: ${g.language}.`,
  ];
  if (g.budgetUSD != null) parts.push(`Budget ~ $${g.budgetUSD.toLocaleString()}.`);
  if (g.teamSize != null) parts.push(`Team size: ~${g.teamSize} people.`);
  if (g.devYears != null) parts.push(`Development time: ~${g.devYears} years.`);
  if (g.source) parts.push(`Source: ${g.source}.`);
  return parts.join(" ");
}

function slugify(s: string): string {
  return s.toLowerCase().replace(/[^a-z0-9]+/g, "-").replace(/(^-|-$)/g, "");
}
```
